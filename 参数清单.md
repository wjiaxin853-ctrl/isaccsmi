对应位置：`Synthetic Data Recorder`

**参数清单**

`Writer`

- `rgb`
  含义：普通彩色图。
  输出：就是正常相机拍到的 RGB 图像。
  常见用途：目标检测、分类、分割、重建、可视化检查。

- `bounding_box_2d_tight`
  含义：目标在图像中的紧贴边界框。
  输出：2D 框坐标，尽量紧贴目标可见区域。
  常见用途：2D 检测训练。

- `bounding_box_2d_loose`
  含义：目标在图像中的宽松边界框。
  输出：2D 框坐标，比 tight 更宽一些，可能包含更多背景。
  常见用途：某些检测或跟踪任务。

- `semantic_segmentation`
  含义：语义分割。
  输出：每个像素属于哪个类别，例如 train、ground、cargo。
  常见用途：语义分割训练。

- `colorize_semantic_segmentation`
  含义：语义分割的彩色可视化图。
  输出：把类别 ID 映射成不同颜色，方便人眼看。
  常见用途：调试、展示，不是最原始训练标签。

- `instance_id_segmentation`
  含义：实例 ID 分割。
  输出：每个像素对应哪个具体实例的 ID。
  常见用途：实例级识别、实例分割。

- `colorize_instance_id_segmentation`
  含义：实例 ID 分割的彩色可视化图。
  输出：不同实例显示成不同颜色。
  常见用途：可视化检查。

- `instance_segmentation`
  含义：实例分割。
  输出：每个像素属于哪个实例目标。
  常见用途：实例分割训练。

- `colorize_instance_segmentation`
  含义：实例分割的彩色可视化图。
  输出：不同实例区域用不同颜色显示。
  常见用途：调试和展示。

- `distance_to_camera`
  含义：每个像素对应点到相机中心的真实欧式距离。
  输出：深度数组。
  常见用途：距离估计、几何分析。
  注意：这不是最典型的 Z-depth，而是点到相机光心的真实距离。

- `distance_to_image_plane`
  含义：每个像素对应点到相机成像平面的垂直距离。
  输出：深度数组。
  常见用途：深度图、重建、视觉算法输入。
  注意：这通常更接近大家说的“标准深度图”。

- `bounding_box_3d`
  含义：目标在三维空间中的 3D 边界框。
  输出：3D 框的位置、尺寸、朝向等。
  常见用途：3D 检测、3D 跟踪。

- `occlusion`
  含义：遮挡信息。
  输出：目标是否被遮挡、遮挡比例等。
  常见用途：筛选样本、评估遮挡条件。

- `normals`
  含义：法线图。
  输出：每个像素对应表面法向量。
  常见用途：几何理解、重建、表面分析。

- `motion_vectors`
  含义：运动向量图。
  输出：像素从上一帧到当前帧的运动偏移。
  常见用途：光流、视频分析、运动估计。

- `camera_params`
  含义：相机参数。
  输出：相机内参、外参、分辨率、矩阵等。
  常见用途：把 2D 图像点还原到 3D、重投影、点云恢复。

- `pointcloud`
  含义：点云。
  输出：由相机视角恢复出来的三维点集合。
  常见用途：3D 重建、配准、点云算法。

- `pointcloud_include_unlabelled`
  含义：点云里是否包含未标注点。
  输出：点云时把没有语义标签的点也保留下来。
  常见用途：需要完整几何时开启，只做有标签训练时可关闭。

- `skeleton_data`
  含义：骨骼数据。
  输出：骨骼节点、关节位置、层级关系等。
  常见用途：人体/动物动作、姿态、骨骼动画相关任务。
  注意：如果场景里没有 skeleton，这项通常没意义。

**常见理解**

- 如果你主要做普通图像任务，最常用的是：
  - `rgb`
  - `bounding_box_2d_tight`
  - `semantic_segmentation`

- 如果你主要做深度和几何任务，最常用的是：
  - `distance_to_image_plane`
  - `distance_to_camera`
  - `camera_params`
  - `pointcloud`

- 如果你主要做 3D 检测，最常用的是：
  - `bounding_box_3d`
  - `camera_params`
  - `distance_to_image_plane`

**补充说明**

- `colorize_*` 一般是给人看，不是最原始训练标签。
- 深度数据本质上是数值，不一定是普通能直接看懂的灰度图。
- 如果深度导出成 `uint16 png`，通常表示的是数值型深度图，不是普通 8-bit 图片。
